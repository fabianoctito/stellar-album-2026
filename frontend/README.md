# stellar-album — frontend (v1)

A minimal Vite + React + TypeScript app for the **core loop**: connect → claim → buy → open → reveal. See [`../docs/frontend-and-testnet.md`](../docs/frontend-and-testnet.md).

> **Status: scaffold, not yet run.** It was written against the documented
> `@stellar/stellar-sdk` (v13) and Stellar Wallets Kit APIs but has not been
> compiled or run end-to-end yet — expect to iterate on small API details
> (client constructor options, arg names, result access) on first run.

## Prerequisites

- Stellar CLI 26+, Node 20+, the Freighter browser extension (set to **Testnet**).
- The 7 contracts deployed on testnet — run from the repo root:
  ```bash
  make bootstrap        # deploys + wires all 7, writes frontend/.env.local
  ```

## Run

```bash
cd frontend
npm install
npm run bindings        # generates typed clients into src/contracts/* from .env.local
npm run dev             # http://localhost:5173
```

## What it does (v1 core loop)

1. **Connect** — Freighter via Wallets Kit; the account is auto-funded via friendbot if new (you never touch XLM — decision D20).
2. **Claim** — Faucet mints your first 1000 ⭐ (then 100 on cooldown).
3. **Buy pack** — Store takes 100 ⭐, gives you a sealed pack.
4. **Open pack** — `Pack.open` burns it and returns the 3 drawn sticker types; they render with rarity (common / rare / legendary).

The spectrum is visible: ⭐ is shown as a **fungible balance** (a number), stickers as **identifiable cards** with a name + rarity.

## Notes / known follow-ups

- **Bindings paths:** `src/contracts/*` are generated by `npm run bindings` and gitignored. `src/lib/clients.ts` imports `Client` from each — if the generated package's entry differs, adjust the import path.
- **Arg names** in calls (`account`, `claimer`, `buyer`, `opener`, `owner`) mirror the contract `__constructor`/fn params; if a binding expects different keys, fix in `App.tsx` / `clients.ts`.
- **Reveal choreography** (rip animation, face-down hold, per-card flip) is deliberately deferred — correct data first (decision D21 / Nicole). The current reveal just renders the 3 cards.
- **v2:** Album paste + Escrow trade screens (contracts already deployed).
- **Fees:** friendbot for v1; swap to Launchtube / OZ relay for mainnet (all submission is via the clients' `signAndSend`, easy to redirect).
